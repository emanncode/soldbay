## 2. Fix hydration mismatch in join form

### Problem

Browser hydration warnings on `/join/buyer` showed className mismatches between server and client rendered `<input>` elements. The `Input` UI component's base classes (`h-10`, `border-none`, `rounded-2xl`, etc.) conflicted with the form-specific `fieldClass` (`h-12`, `border`, `rounded-xl`, etc.) when merged via `cn()` + `tailwind-merge`. The merge produced different class output between server and client bundles.

### Fix

Replaced the three `<Input>` usages in `join-form.tsx` with plain `<input>` elements using `fieldClass` directly, bypassing the `cn()` merge entirely. Also added explicit `type="text"` to the name and sellCategory inputs (fixing a `type={undefined}` issue).

### Files modified

| File | Change |
|---|---|
| `src/components/join-form.tsx` | Replaced `<Input>` with `<input>` for name, email, and sellCategory fields; removed unused `Input` import; added explicit `type` props |
