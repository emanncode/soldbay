| File | Change |
|---|---|
| `src/middleware.ts` | Either/or auth on seller-only routes (`POST /api/listings`, `/seller/*`): (1) if `Authorization: Bearer …` present → verify mobile JWT → **401** invalid/expired, **403** valid but `role !== SELLER`, else allow; (2) else existing NextAuth cookie session path unchanged (**401** / **403** same as before). Web and mobile do not replace each other. |
| `package.json` / `package-lock.json` | Added `jsonwebtoken`, `@types/jsonwebtoken`, `jose` |

### Design notes

- **Sign:** `jsonwebtoken` in Node route handler only (`sign-mobile-token.ts`).
- **Verify in middleware:** Next.js middleware is **Edge**, so `jsonwebtoken` cannot be imported there. Split modules: middleware imports only `mobile-auth.ts` (**jose**). Tokens remain standard HS256 JWTs signed with `AUTH_SECRET` — compatible.
- **Payload:** `{ userId, role }` (+ `iat` / `exp` from jsonwebtoken). Expiry **30 days**.
- **Untouched:** `src/auth.ts`, `src/auth.config.ts`, web Credentials cookie login, `/login` page (not built), school-email restriction, student ID verification.

## 14. Scroll animations (enter/exit) → then performance fix

### Initial behavior

- Framer Motion `whileInView` with **`once: false`** so sections animated **in on scroll enter** and **out on scroll leave**
- Shared helpers in `src/lib/motion.ts`: `sectionVariants`, `fadeUpVariants`, `scaleInVariants`, `staggerCardVariants`, `scrollViewport`
- Wired through `AnimatedSection` + hero / how-it-works cards / social-proof / FAQ / footer

### Performance issue

Reverse-on-leave (`once: false`) re-triggers JS + paint while scrolling and can make the page feel laggy (especially with glass `backdrop-filter`).

### Fix applied

| Change | Why |
|---|---|
| `scrollViewport.once = true` | Animate **in once only** — no reverse on scroll up |
| Opacity + `translateY` only | Removed scroll-time `scale` (cheaper paint) |
| Shorter durations (~0.4s) | Less frame work |
| `useReducedMotion()` | Skip motion when OS prefers reduced motion |
| Hero uses mount `animate="visible"` | No scroll observer fighting first paint |
